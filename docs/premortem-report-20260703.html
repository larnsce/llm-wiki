Prob MedImpact Critical

④ The hidden assumption (the one to internalize)

That the job is building a correct v2 engine — when the actual success criterion is a knowledge base still used daily. Almost every failure traces to treating the wiki as an abstraction (fixtures, specs, byte-diffs, dual-mode promises) rather than as the maintainer's concrete existing corpus and daily habit. Canon-first, spec-heavy, audit-subagents and a trust layer optimize engine correctness against a multi-user/adversarial threat model that a single trusted author never faces. "Correct engine" and "still used daily" are different targets — and the plan is aimed at the first.

⑤ Revised plan — each maps to a failure mode

  1. Add the missing migration issue (kills #10). A one-time corpus converter + a "grandfather" lint mode (severity floor for pre-v2 pages). Release gate = run wiki-lint against the real vault, not just tests/fixtures.
  2. Demote "byte-identical" to "semantically equivalent" (kills #1). Define a normalization (strip trailing whitespace, sort frontmatter keys, canonical newlines) and diff on that. Never let init_wiki.py block on xxd.
  3. Add a spec-consistency check (kills #2). A crude script that greps the lint-rule count, property enums, and the reliability rule across specs + wiki-core references + both Schema pages and fails CI if they disagree. Makes the DRY invariant actually enforceable.
  4. Batch the ingest checkpoint (kills #5). One review for the whole queue with per-source expand — not a mandatory stop per paper. Log --auto share; if it exceeds 50%, the design failed, not the user.
  5. Golden-transcript tests for LLM behavior (kills #6). Pin a fixed source → expected cite:: targets + audit verdicts; re-run on every prompt/model change. Cheap; catches silent knowledge regression the mechanical harness can't.
  6. Reframe the secret gate as an assist, not a guarantee (kills #7). Rewrite the regex for raw source bytes (drop the :: assumption), add email/national-ID PII patterns (advisory), and make "clean" output say "scanned N patterns — NOT a guarantee; eyeball sensitive sources." Track sensitive_source_types entries added right after a block event (bypass signal).
  7. Pick a tier-1 mode (kills #8). Logseq (the dogfooded one) is supported; label Obsidian experimental unless CI instantiates a real Obsidian vault and byte-diffs + round-trips cite::. Don't promise dual-mode the test matrix never covers.
  8. Define a minimal v2.0.0, defer the rest (kills #3, #9). Monolith → skills + core scripts + one mode is a coherent release. Push audit/update/citations to v2.1 if momentum flags. Ship a smaller whole thing, never a bigger half thing.

⑥ Kill criteria — decide now, before emotion

  • If >3 weeks pass between merged implementation PRs after I-4 → freeze scope: ship what's coherent as v2.0.0, defer the rest. Do not sit in half-migrated limbo.
  • If the init_wiki.py byte-identical test is red >3 days on whitespace/ordering deltas → abandon byte-identical for semantic-equivalent that day.
  • If the first wiki-lint run on the REAL vault returns >~20 violations with no grandfather path → do not tag v2.0.0; build migration first.
  • If --auto is in >50% of ingests in month 1 → the checkpoint failed as designed; redesign to batch review before claiming "human in the loop" closes #4.

⑦ Pre-launch checklist

  • ☐ Run wiki-lint against the actual production vault (not fixtures); confirm a migration/grandfather path and a manageable violation count.
  • ☐ Instantiate a real Obsidian vault from init_wiki.py; byte-diff output and round-trip a cite:: — or downgrade Obsidian to experimental in the README.
  • ☐ Wire the spec-consistency check (rule counts, enums, reliability rule agree across all narrations) into CI.
  • ☐ Add ≥1 golden-transcript test for ingest citations and audit verdicts; run it on prompt/model changes.
  • ☐ Confirm secret_scan.py catches a planted email / national-ID / PDF-embedded secret (not just a token:: line), and that its "clean" message disclaims guarantee.
  • ☐ Verify the cutover is atomic: main never ships with wiki.md deleted but a workflow it provided only half-ported.

01

The "byte-identical" trap stalls the critical path

Prob MedImpact Med

Story. wikilib.py lands fast, but init_wiki.py stalls the moment "byte-identical to setup.sh for default answers" meets reality: heredoc trailing newlines Python's write() omits; a sed tab vs four spaces; Obsidian frontmatter key ordering (string-concat vs dict). Each one-line fix surfaces another. The tip was keeping byte-identical a hard gate — four evenings diffing xxd instead of writing REQs — across every default-answer permutation × two editors. Since #12 is the shared import for every downstream script, I-5…I-11 can't even scaffold. The monolith keeps shipping because it still works.

Assumption: that "byte-identical" was a cheap mechanical check, not a spec of every incidental shell-script artifact — whitespace, ordering, newlines — no one had canonized.

Early signs: the init test is red >3 days with the diff shrunk to whitespace/newline/key-order only; downstream issues have zero commits because nothing can import a not-yet-frozen wikilib.py.

02

"No rule in more than one place" is an unenforceable manual invariant

Prob HighImpact Med

Story. The invariant is an acceptance criterion backed by nothing that runs. One rule (reliability aggregation) must appear in the spec, in wiki-core references, and — because users read them directly — in both template Schema pages, which already diverge line-for-line today. "Load by reference" fixes code duplication, not prose: a Logseq Schema page can't @import a reference. It tips at ~I-8 when a property enum gains a value, updated everywhere but obsidian/Schema.md. Nothing compares the five narrations; the only check was the maintainer re-reading 8 SKILL.md + 8 specs + 2 Schema pages every PR — which a solo dev stops by week three. The MAY-vs-SHALL contradictions v2 existed to kill get respread across more files.

Assumption: a convention can live in one place while five audiences each need to read it in situ, with no tool to detect when the copies disagree.

Early signs: diff of the two Schema.md pages shows normative divergence (already true today); the stated lint-rule count differs across SKILL.md, spec, and docs.

03

Scope collapse into half-migrated limbo

Prob HighImpact High

Story. I-4 is done carefully and slowly. Waves 1–2 feel like progress on weekends, but I-4 forces the deferred decision that every script behave identically in both modes — silently doubling every downstream issue. By I-6, each "one issue" is two implementations plus a diff test, and a day-job crunch eats three weekends. Momentum dies exactly where the chain fans out: I-6 gates I-7, I-9, I-10 — four issues freeze at once. The repo is now bimodal: wiki.md still works and is used daily, while skills/ holds half-ported workflows the merged specs describe as finished. A README-follower hits a citation feature (I-9) documented but never built. Bad-options fork: finish six more chained two-mode issues on nights-and-weekends, or revert and admit the specs lied. Neither happens.

Assumption: a solo maintainer's spare-time velocity is roughly constant across 12 issues — when per-issue cost compounds through the two-mode requirement and the dependency chain.

Early signs: >3 weeks between merged implementation PRs after I-4; main still ships wiki.md while dev has a non-empty skills/ tree — both present past the I-6 mark.

04

Spec-implementation drift — canon-first honored in the breach

Prob HighImpact High

Story. REQ-904 ("source-file:: = union of the page's cite:: refs") is mechanically checkable on paper, but in Logseq source-file:: is page-level while cite:: is per-block, and the outliner reorders/merges blocks under you — constant invariant violations on legitimate edits, so the gate gets downgraded from enforce to warn. REQ-922 ("one verification subagent per cited source, in parallel") fans a 30-cite page into 30 subagents — slow, expensive, rate-limited — so sources get batched "temporarily," breaking the isolation guarantee. REQ-951/952's mandatory source + before/after diff fight batch use, so an --auto path grows that skips the confirmation the spec makes unconditional. By tag, #20's "every doc claim traceable to a REQ" is true only textually — claims trace to REQs the code no longer honors.

Assumption: a design's correctness can be settled on paper against Logseq's block model and subagent economics before a single line of code tests it.

Early signs: any REQ downgraded SHALL/gate → "warn"/skipped in a commit or config flag; citations/audit/update specs with zero edits after implementation commits touch the same behavior.

05

The mandatory ingest checkpoint reintroduces the friction it replaced

Prob HighImpact Med

Story. The first few ingests, the maintainer actually stops — reads the plan, the reliability ratings, the contradictions, answers "what to emphasize." It feels like the gist restored. Then a conference dump lands: eighteen papers. Stopping eighteen times to read near-identical analysis blocks is unbearable, so --auto "just for this batch." It works fine — that's the poison: a skipped checkpoint's cost (a bad reliability call, a missed contradiction) is silent and deferred, while the checkpoint's cost is immediate and certain. Every rational ingest optimizes toward --auto. By month two it's muscle memory; #14 shipped as specified and #4 is behaviorally reopened, now with a flag that launders the autonomy as a choice. The other branch is worse: those who don't reach for --auto simply ingest less, and the wiki stops growing.

Assumption: a checkpoint made mandatory in code is also mandatory in behavior — that an always-available skip won't become the default when review cost is immediate and skip cost is invisible.

Early signs: --auto in >50% of ingest invocations within month one; median checkpoint dwell time trending toward zero (displayed and dismissed in <3s, no input entered).

06

The test harness covers only the mechanical shell

Prob MedImpact High

Story. A line gets drawn — "the LLM-in-the-loop part cannot be asserted by a script" — and everything past it (synthesis quality, cite:: correctness, the supported/partial/unsupported verdict) is demoted to a manual protocol "executed once," by the person who wrote it, on clean fixtures. That single run is the entire regression net for the tool's only real value. test_pipeline.sh goes green on orphans, broken refs, the fake secret — all true, all cheap, all beside the point. Then routine maintenance: a prompt tweak plus a model bump, both green, because nothing reads whether a cite:: points at a claim the source actually makes. Three months on it surfaces in the maintainer's own wiki — pages synthesized from sources that don't support them — discovered by hand, in the one place trusted by definition, where wrong content reads as fact and compounds across links.

Assumption: because the LLM behaviors can't be asserted mechanically, they need no regression net at all — a one-time manual protocol substitutes for continuous testing.

Early signs: a commit touching prompts / ingest / audit merges with only test_pipeline.sh green and no re-run of the manual protocol logged; a golden-transcript check on a fixed source drifts after a prompt/model change.

07

The secret gate is ~70% effective — and that's worse than none

Prob MedImpact Critical

Story. secret_scan.py reuses REQ-042's regex — written for synthesized pages: token::, password::, 40+ char base64. The :: are Logseq delimiters that never appear in a Zotero export or PDF text, so on real source bytes it matches almost nothing except the base64 rule, which fires on every embedded image and font subset. Week two, five false positives on innocuous PDFs; the maintainer routes them into sensitive_source_types and internalizes "it cries wolf." Then the case that matters: a Zotero library with author emails and a scanned consent form (national IDs in binary PDF text the strings pass fragments across object boundaries) sails through clean. The gate's reassuring silence retires the manual eyeballing, and the source moves verbatim into git-tracked ingested/. #6's sticky-exposure gap is realized — worse, because a green check certified it safe.

Assumption: a partial regex that catches some credentials is a net safety gain — when any automated "clean" transfers responsibility off the human without earning it.

Early signs: sensitive_source_types grows entries added right after a blocked-ingest event (bypasses, not policy); zero true-positive blocks over many ingests while sources demonstrably contain emails/IDs.

08

The two-tool-mode tax — the non-dogfooded mode silently breaks

Prob HighImpact Med

Story. The two Schema templates aren't one file with a variable — they're two hand-maintained files with different structure. That's four template pairs plus init_wiki.py plus wikilib.py plus every script carrying an if mode == logseq branch. The maintainer dogfoods Logseq. At I-9 the cite:: split ("Logseq block property; Obsidian indented child bullet") means the Logseq path runs daily while the Obsidian child-bullet path is written once, eyeballed against a hand-made fixture, and never round-tripped through a real vault. "Verified in both modes" collapses to "same author wrote both tests with the same blind spot." Lint globs Wiki___*.md vs Wiki/**/*.md; the Obsidian nested glob matches zero files in some layouts, so lint reports green by finding nothing. Three months on, the first real Obsidian user hits byte-wrong pages, cite:: children that don't parse, lint false-passing on an empty set.

Assumption: a solo maintainer can keep two divergent format implementations equally correct while only ever running one of them.

Early signs: Logseq commits outnumber Obsidian ~5:1, no CI job instantiates a real Obsidian vault and byte-diffs; lint on an Obsidian vault exits 0 while processing 0 files.

09

Over-engineering for an audience of one

Prob Med–HighImpact High

Story. Building the openspec canon turns every real change — a citation tweak, a cache-eviction change — into a three-surface edit (REQ + wiki-core reference + 8 skills). The maintainer notices he spends Saturdays servicing the spec instead of writing notes. The audit subsystem with parallel subagents and the L1/L2 hub-index cache with LRU-demote are built for scale and adversarial trust a single-user personal wiki never experiences — provenance/trust guards a threat model of one trusted person. By month two, adding a note sometimes trips a stale REQ or a skill assuming the old cache contract; capturing a thought costs a debugging detour. The daily-use metric inverts: the wiki gets opened to fix the wiki, not to think. Upstream Karpathy dormancy removes the external pull. v2.0.0 gets tagged — it just isn't used; the architecture outlived its user's patience.

Assumption: robustness machinery justified by multi-user/adversarial scale pays off for a single-author personal wiki — when its only cost, maintenance, is borne by that same single person.

Early signs: commits touching skills/specs/scripts vs commits adding actual wiki content trend above ~1:1 over a month; days-since-last-note grows while days-since-last-architecture-edit stays near zero.

10

What the others missed — no migration path for the EXISTING corpus

Prob HighImpact High

Story. All 12 issues build the machinery that processes a wiki; none touches the hundreds of pages the maintainer already edits daily. Those were authored under v1 conventions — prose citations, old block structure, no schema discipline. The moment v2 ships, wiki-lint's 12 rules fire against real content and return a wall of violations; wiki-query and wiki-audit assume block-native cite:: the existing corpus doesn't have. There's no migration issue, no converter, no grandfather escape hatch — and the harness runs on clean synthetic fixtures, so this never surfaces before tag. The maintainer, whose entire reason for v2 was to keep using his own knowledge base, now faces a manual reformatting slog across the whole vault. Realistically he reverts to v1 on the real wiki (v2 becomes shelfware validated only against fixtures) or the daily-use loop breaks — monolith gone, but the actual wiki left behind.

Why overlooked: the list scrutinized the code-and-spec pipeline exhaustively but treated the wiki as an abstraction, never as a concrete pre-existing pile of the maintainer's own bytes that must survive cutover.
Assumption: building a correct v2 engine is the whole job — silently assuming existing content will conform, or that green fixtures mean the real vault works.

Early signs: no issue/script/REQ mentions converting existing pages; all fixtures are freshly authored. First wiki-lint on the real vault returns dozens-to-hundreds of violations.
