Premortem: Glossary (v2.3) + Personal Pipeline (v3.0)

Frame: it is January 2027, six months on. Both additions to llm-wiki have failed. Ten investigators worked backward to explain why. Plan under test: docs/roadmap-glossary-personal-pipeline.md. Full transcript: prompts/2026-07-04-001-premortem-glossary-personal-pipeline.md.

Synthesis

Most likely failure#8 Tooling outruns behavior

Agents can ship all 13 issues in days while the graph is still empty; five ceremony-heavy habits (namespace placement, provenance, checkpoints, glossary curation, daily review) land on day one of a cold start. The daily review, which the setup plan itself calls the thing that decides survival, dies at week two. This failure also amplifies #1 (curation), #3 (import flood), and #5 (synthesis rot): every unformed habit turns a designed safeguard into noise.

Most dangerous failure#7 Third-party privacy leak

A candid sentence about a named student passes the shape-based scanner, gets confirmed as row 7 of 12 at 7am, enters a git-committed people page, and later surfaces in a shared draft. Harm to a real person, GDPR exposure, and git history makes it near-permanent.

Most irreversible failure#10 archive.db loss

The one file declared irreplaceable is also the one file git refuses to cover, with no backup REQ anywhere in the plan. One git clean -xfd by an agent (the likeliest executor) destroys it and dangles every committed provenance link forever.

The hidden assumption"Because implementation is free, shipping the tool is progress."

The system's real products are habits and decisions: the filled Rule column, the daily review, the curation pass. The plan engineers the free part (code) and assumes the expensive part (behavior) follows. Its corollary runs through half the failures: raw capture (voice transcripts, imported termbases) keeps getting treated as if it had the trust status of curated, decided content. A transcript is not a source; an unreviewed staging row is not a decision.

The revised planTwelve binding changes (full text in the roadmap)

  1. Base loop first: no new issue is filed until the fresh graph survives two weeks of real use of the EXISTING suite. (#8)
  2. Glossary starts by hand (G-0): templates + workflow doc only; the spec/lint/skill layer (G-1..G-6) is built only after 20+ hand-decided Rule rows exist. (#1, #8)
  3. Phase gates become merge gates: P-3 blocked on a recorded Phase 0 exit test; P-4/P-5 blocked on three weeks of living daily review. (#8)
  4. G-4 deleted: no hand-rolled YAML parser; the agent transforms glossary.yml at import time and verifies counts against the source. (#9)
  5. Import is pull, not bulk: staging holds only captured/requested terms; drafting context loads domain pages only, never staging. (#3)
  6. Voice defaults to journal-only: wiki writes are per-row opt-in; people-page rows always shown in full and confirmed individually; assessments of people (health, family, grades, conflicts) are never promoted at all. (#5, #7)
  7. Voice provenance = capture-backed: reliability low by default, a distinct audit verdict class; upgrading requires a real source. (#5)
  8. Namespace contract becomes a parsed canon surface plus a grep gate before REQ-960 is ever amended. (#2)
  9. Dead-man status line opens every journal summary (inbox age, unprocessed count, rebuild age) plus a weekly canary memo. (#4)
  10. archive.db durability as spec REQ: nightly off-machine copy, quarterly restore drill, provenance-resolution audit check, and a vault-level ban on git clean -xfd for agents. (#10)
  11. index.db provably derived: frozen 3-table schema, no pre-commit hooks, rebuild-reproducibility asserted in the harness, nothing without a markdown source. (#6)
  12. Personal tier opt-in: voice/storage ship behind setup.sh --with-personal, labeled separately in the README, synthetic fixtures only. (repo identity)

Kill criteriaDecide now, not in the moment

  • Phase 0 exit test not passed within two weekends of P-2: stop filing v3.0 issues.
  • Fewer than 20 hand-decided Rule rows after 4 weeks of G-0: do not build the glossary tooling; it stays a manual convention.
  • Daily review skipped 5 consecutive days in the first three weeks: pause all expansion, shrink ceremony to journal-only; do not add tooling to fix a habit problem.
  • Two voice-sourced wiki claims wrong within one month: voice drops to journal-only until the checkpoint is redesigned and re-validated.

Pre-launch checklistBefore the respective first use

  • Two weeks of real base-loop use recorded (gate for any new issue)
  • archive.db backup + one restore drill executed before the first real voice note
  • Dead-man status line + canary procedure live when P-3 merges
  • Per-row people-page confirmation + sensitive-content rule in the P-3 checkpoint
  • check_canon namespace surface + grep gate in the same PR that amends REQ-960
  • Pull-import and staging-never-as-context rules in the workflow doc from day one

The failure modes

#1 Glossary curation loop dies likely / medium

Capture costs two seconds; curation costs a 40-minute session that feels like grading and has no trigger, no cost of skipping, no exit test. 60+ todos by September, a 12-row domain page by October; drafting stops loading glossary context; posts ship with three renderings of "notebook".

Underlying assumption

Friction-free capture implies the curation pass will happen; only capture was engineered.

Early warnings

Open #glossary-todo count rises 3+ weeks while decided rows stay flat; zero skill invocations in a month.

#2 Fourth-namespace spec churn medium / high

"Exactly three namespaces" lives as prose in 7+ surfaces (spec title, Scenario 7, checklists, scope guard, roadmaps, templates). G-1 amends the REQ; the narration diverges; check_canon stays green because prose counts are narration, not structure. The canon's green checkmark stops meaning anything.

Underlying assumption

check_canon covers every surface where the contract is narrated; it only covers counted structure.

Early warnings

grep for "three namespaces" nonzero after G-1 while check_canon is green; G-1 diff touches fewer files than the grep returns.

#3 Import flood likely / medium

400 Glosario rows land in week one and become the largest page in an empty graph. Lookup costs zero; promotion costs three edits plus the one thing that cannot be automated, the Rule decision. Staging becomes the de facto glossary; published texts encode Glosario's choices, not the maintainer's.

Underlying assumption

A complete third-party table next to an empty curated one motivates curation rather than substituting for it.

Early warnings

Staging page loaded as drafting context more often than any domain page; promoted-terms count flat two weekly reviews running.

#4 Silent plumbing decay likely / medium

An iOS update silently kills the Shortcuts automation; memos strand on the phone, invisible to every Mac-side component. Summaries keep flowing from backlog, so the loop looks alive while dead. The eventual 40-memo catch-up batch is worthless, and the habit dies of it.

Underlying assumption

Every failure queues data where the drain loop can see it; silence means empty, never broken.

Early warnings

Newest inbox file older than 3 days while the phone shows recent memos; five straight days of summaries covering zero new rows.

#5 Synthesis quality rot likely / high

"Marek" transcribes as "Mark" and lands on the wrong person page; "I should maybe" becomes "Planned:" on a project page. Provenance points at transcript rows nobody rereads, so wiki-audit verifies mush against mush. One correction from the real Marek later, the checkpoint flips to reject-everything and the voice leg gets commented out.

Underlying assumption

A transcript row is a source in the v2 sense; it is unvetted raw capture wearing a citation.

Early warnings

Approval rate over ~90% with under 10s median review; zero provenance links ever opened in month one.

#6 Second plane eats the architecture medium / high

The pre-commit rebuild gets bypassed with --no-verify; the staleness warning gets muted; a SQL-fluent maintainer adds one table, then an importer that bypasses the vault. index.db stops being derived, rebuilds become scary, SQL contradicts pages, and some facts live only in a gitignored binary.

Underlying assumption

"Derived and disposable" is a property kept by design rather than a discipline re-earned at every schema temptation.

Early warnings

Any --no-verify commit or rebuild over ~10s; first table whose data has no markdown source (rebuild+diff no longer reproduces).

#7 Privacy leak through the seam less likely / severe

"Student M admitted she's failing because of the situation at home" is just German prose about a named person: no pattern matches it. Confirmed on autopilot as row 7 of 12 at 6:58am, it enters a git-committed people page and later a shared draft preview. A colleague recognizes the student. Git history makes it permanent.

Underlying assumption

Dangerous content is pattern-detectable, and a daily confirm step constitutes review.

Early warnings

Sub-30s approvals of 10+ row batches with zero rejections for 14 days; people-page bullets naming health/family/grades/conflict with no flag.

#8 Tooling outruns behavior most likely / high

All 13 issues merge within days because building costs one prompt; the Phase 0 exit test never runs because the gated thing already exists. The first real morning demands five simultaneous habits on an empty graph; review takes 25 minutes instead of 5-10; by day 9 it gets skipped, and by week two the system is a museum: 108 green assertions, 14 pages.

Underlying assumption

Shipping capacity and adoption pacing are independent, so free implementation cannot hurt a habit plan.

Early warnings

Any week where merged PRs outnumber new graph pages; P-4/P-5 merged before a recorded Phase 0 pass.

#9 Hand-rolled YAML parser medium / low-medium

The parser silently drops ~80 multi-line and quoted entries on day one (including most German ones), breaks loudly when Glosario reorders keys, and becomes a recurring chore; the exhausted fix is "import yaml, just this once", cracking the zero-dependency rule's authority for every future script.

Underlying assumption

Glosario's YAML is a stable flat subset, rather than real YAML maintained by someone else.

Early warnings

Imported rows below the source's term count on day one; any post-ship commit touching the importer.

#10 archive.db loss counter-check / irreversible

Markdown is committed; index.db is rebuildable; archive.db is gitignored AND irreplaceable, with no backup REQ in any milestone. A fresh clone, a disk failure, or an agent running git clean -xfd deletes months of voice notes; index.db rebuilds cleanly, masking the loss; every committed provenance id dangles forever.

Underlying assumption

"Gitignored plus irreplaceable" was written as a merge-safety rule; nobody asked what replaces git as the durability mechanism.

Early warnings

The spec ships with no backup/restore REQ and no provenance-resolution check; a cleanup or migration is declared healthy without verifying archive.db row counts.

9 parallel investigators (one per failure reason, isolated from each other) + 1 adversarial counter-check, which contributed #10. All ten produced a failure story, an underlying assumption, and observable early warnings.
